# 在线拍卖平台用户文档

## 1. 系统概述

在线拍卖平台是一个面向校园或小型社区的 Web 拍卖系统，支持物品发布、审核、竞价、支付、评价等完整业务闭环。用户可以通过浏览器访问系统，参与拍卖活动。

### 1.1 系统功能

- 用户注册与登录
- 物品发布与审核
- 拍卖活动管理
- 实时竞价与通知
- 订单与支付结算
- 评价与反馈
- 统计分析与报表

### 1.2 访问方式

- 前端地址：`http://127.0.0.1:3000`（Next.js 开发服务器）
- 后端 API：`http://127.0.0.1:18080`（Drogon 应用服务器）
- 健康检查：`http://127.0.0.1:18080/healthz`

## 2. 用户角色

系统包含以下用户角色：

| 角色 | 权限说明 |
|------|----------|
| 普通用户 | 可注册账号、发布物品、参与竞价、支付订单、提交评价 |
| 管理员 | 审核物品、创建拍卖活动、查看统计报表、处理异常订单 |
| 运维/客服 | 查看系统日志、查询交易记录、处理用户咨询 |

## 4. 操作流程

### 4.1 用户注册与登录

#### 4.1.1 注册新账号

1. 访问系统首页 `http://127.0.0.1:3000`
2. 点击右上角"登录"按钮
3. 在登录页面点击"注册"标签
4. 填写用户名、密码、手机号
5. 点击"注册"按钮
6. 注册成功后自动跳转到拍卖大厅

#### 4.1.2 登录系统

1. 访问系统首页 `http://127.0.0.1:3000`
2. 点击右上角"登录"按钮
3. 输入用户名和密码
4. 点击"登录"按钮
5. 登录成功后跳转到拍卖大厅

#### 4.1.3 登出系统

1. 点击页面右上角用户头像
2. 选择"登出"选项
3. 系统清除登录状态，返回首页

### 4.2 卖家操作流程

#### 4.2.1 发布物品

1. 登录系统后，点击导航栏"发布拍品"
2. 进入发布向导页面，分为 4 个步骤：

**步骤 1：描述拍品**
- 填写拍品标题（必填）
- 填写拍品描述（必填）
- 选择分类（数码 3C、运动装备、图书教材、生活闲置）
- 设置起拍价（必填）

**步骤 2：上传实拍**
- 点击上传区域或拖拽图片到上传池
- 第一张图片自动设为封面首图
- 可上传多张图片

**步骤 3：制定竞价契约**
- 设置最小加价幅度
- 设置拍卖开始时间
- 设置拍卖结束时间
- 配置延时保护规则（可选）

**步骤 4：检查并上架**
- 预览所有填写信息
- 确认无误后点击"提交审核"
- 系统自动保存草稿到本地

#### 4.2.2 查看审核状态

1. 登录系统后，进入"我的拍品"页面
2. 查看拍品列表及审核状态：
   - 草稿：尚未提交审核
   - 待审核：已提交，等待管理员审核
   - 审核通过：可参与拍卖
   - 审核拒绝：可查看驳回原因并修改后重新提交

#### 4.2.3 确认发货

1. 当拍卖结束且买家支付成功后，卖家会收到通知
2. 进入"我的订单"页面
3. 找到待发货订单
4. 点击"确认发货"按钮
5. 系统更新订单状态为"待收货"

#### 4.2.4 评价买家

1. 当买家确认收货后，卖家可对买家进行评价
2. 进入"我的订单"页面
3. 找到已完成订单
4. 点击"评价"按钮
5. 填写评分（1-5 星）和评价内容
6. 提交评价

### 4.3 买家操作流程

#### 4.3.1 浏览拍卖列表

1. 登录系统后，进入拍卖大厅 `/auction/hall`
2. 使用左侧筛选面板：
   - 价格区间筛选
   - 卖家信用分筛选
   - 交易方式筛选
3. 右侧显示拍品卡片矩阵
4. 点击拍品卡片进入详情页

#### 4.3.2 查看拍品详情

1. 在拍卖大厅点击拍品卡片
2. 进入竞价详情页 `/auction/detail/[id]`
3. 页面分为左右两栏：
   - 左侧：高清图片、描述、参数、卖家评价
   - 右侧：竞价控制台、当前最高价、出价历史

#### 4.3.3 参与竞价

1. 在拍品详情页查看当前最高价
2. 确认出价金额（必须高于当前价 + 最小加价幅度）
3. 点击快捷加价按钮或输入自定义金额
4. 点击"PLACE BID"按钮
5. 系统处理出价请求：
   - 成功：出价记录变为绿色，显示最新价格
   - 409 冲突：价格已被其他买家抢先更新，需刷新后重新出价
   - 429 限流：出价过于频繁，需等待冷却时间

#### 4.3.4 实时价格更新

系统支持两种实时更新方式：

1. **WebSocket 实时推送**（优先）
   - 页面自动建立 WebSocket 连接
   - 实时接收价格变化、拍卖延时、被超越提醒
   - 连接成功时显示绿色状态指示

2. **轮询降级**（WebSocket 不可用时）
   - 每 4 秒自动刷新价格
   - 页面顶部显示降级横幅提示

#### 4.3.5 支付订单

1. 拍卖结束后，获胜买家会收到待支付通知
2. 进入支付页 `/checkout/[orderId]`
3. 查看订单详情：
   - 落锤价（成交金额）
   - 管理费（如有）
   - 总计金额
4. 选择支付方式：
   - 校园一卡通
   - 微信/支付宝模拟钱包
5. 点击"确认支付"按钮
6. 系统处理支付：
   - 支付中：显示进度条
   - 支付成功：显示绿色对勾
   - 支付失败：显示错误信息

#### 4.3.6 确认收货

1. 当卖家发货后，买家会收到通知
2. 进入"我的订单"页面
3. 找到待收货订单
4. 点击"确认收货"按钮
5. 系统更新订单状态为"已完成"

#### 4.3.7 评价卖家

1. 当订单完成后，买家可对卖家进行评价
2. 进入"我的订单"页面
3. 找到已完成订单
4. 点击"评价"按钮
5. 填写评分（1-5 星）和评价内容
6. 提交评价

### 4.4 管理员操作流程

#### 4.4.1 审核物品

1. 使用管理员账号登录系统
2. 进入管理大盘 `/admin/dashboard`
3. 在"待审核"标签页查看待审物品列表
4. 点击物品行，右侧滑出审核抽屉
5. 查看物品详情、图片、描述
6. 执行审核操作：
   - 点击"批准"按钮：物品状态变为"审核通过"
   - 点击"驳回"按钮：填写驳回原因后提交

#### 4.4.2 查看统计报表

1. 在管理大盘 `/admin/dashboard`
2. 查看 KPI 卡片：
   - 今日成交额
   - 今日出价次数
   - 活跃拍卖数
   - 待处理订单数
3. 查看日报统计：
   - 选择日期范围
   - 系统显示指定时间段的统计数据

#### 4.4.3 处理异常订单

1. 在管理大盘查看异常订单列表
2. 点击订单查看详情
3. 根据情况进行处理：
   - 标记异常
   - 手动关闭订单
   - 发起补偿流程

### 4.5 运维/客服操作流程

#### 4.5.1 查看系统日志

1. 使用运维账号登录系统
2. 进入管理大盘
3. 查看操作日志：
   - 用户登录/登出记录
   - 管理员审核记录
   - 支付处理记录
4. 查看任务日志：
   - 拍卖结束任务
   - 统计汇总任务
   - 补偿任务

#### 4.5.2 查询交易记录

1. 在管理大盘使用搜索功能
2. 输入订单号、用户 ID 或拍卖 ID
3. 查看交易详情：
   - 出价历史
   - 订单状态
   - 支付记录

#### 4.5.3 处理用户咨询

1. 查看用户通知记录
2. 根据用户问题进行处理
3. 必要时标记异常并提交管理员

## 5. 常见问题

### 5.1 登录相关

**Q: 忘记密码怎么办？**
A: 当前系统不支持自助重置密码，请联系管理员重置。

**Q: 登录时提示"账号已被冻结"？**
A: 账号因违规操作被管理员冻结，请联系客服解冻。

**Q: 登录时提示"账号已被禁用"？**
A: 账号因严重违规被永久禁用，无法恢复。

### 5.2 竞价相关

**Q: 出价时提示"金额不合法"？**
A: 出价金额必须高于当前最高价 + 最小加价幅度，请检查后重新出价。

**Q: 出价时提示"拍卖已结束"？**
A: 该拍卖活动已结束，无法继续出价。

**Q: 出价时提示"出价冲突"（409）？**
A: 其他买家已抢先出价，当前价格已更新，请刷新页面后重新出价。

**Q: 出价时提示"操作过于频繁"（429）？**
A: 出价频率过高，请等待冷却时间后再试。

**Q: 页面顶部显示"实时通道已断开"？**
A: WebSocket 连接失败，系统已自动降级为轮询模式，价格更新可能有延迟。

### 5.3 支付相关

**Q: 支付时提示"支付失败"？**
A: 可能是支付渠道异常，请稍后重试或更换支付方式。

**Q: 支付成功但订单状态未更新？**
A: 支付回调可能存在延迟，请刷新页面或稍后查看。

**Q: 超时未支付会怎样？**
A: 超时未支付的订单将自动关闭，拍品可能重新上架。

### 5.4 发布相关

**Q: 提交审核时提示"至少需要一张图片"？**
A: 请上传至少一张拍品图片后再提交审核。

**Q: 审核被驳回怎么办？**
A: 查看驳回原因，修改拍品信息后重新提交审核。

**Q: 草稿丢失了？**
A: 草稿保存在浏览器本地存储中，清除浏览器数据会导致草稿丢失。

### 5.5 通知相关

**Q: 没有收到通知？**
A: 请检查：
1. 是否已登录系统
2. 浏览器是否允许通知
3. 网络连接是否正常

**Q: 如何查看历史通知？**
A: 点击页面右上角铃铛图标，查看通知中心。

## 6. 系统限制

### 6.1 性能限制

- 普通页面请求响应时间 ≤ 2 秒
- 出价接口平均响应时间 ≤ 500 毫秒
- 支持约 200 个并发在线用户

### 6.2 业务限制

- 单个用户最多同时发布 50 个拍品
- 单个拍品最多上传 10 张图片
- 出价金额最大为 999,999.99 元
- 拍卖持续时间最长为 30 天

### 6.3 安全限制

- 密码长度 6-50 个字符
- 登录令牌有效期 120 分钟
- 出价频率限制：1 秒内最多 5 次

## 8. 附录

### 8.1 状态说明

**拍品状态：**
- 草稿：尚未提交审核
- 待审核：已提交，等待管理员审核
- 审核通过：可参与拍卖
- 审核拒绝：需修改后重新提交
- 待拍卖：审核通过，等待拍卖开始
- 竞拍中：拍卖进行中
- 已成交：拍卖结束，有人出价
- 已流拍：拍卖结束，无人出价
- 交易完成：买家支付，卖家发货，交易结束

**订单状态：**
- 待支付：拍卖结束，等待买家支付
- 已支付：买家支付成功
- 待发货：卖家待处理
- 待收货：卖家已发货，等待买家确认
- 已完成：买家确认收货
- 已关闭：超时未支付或取消
- 已评价：双方完成评价
