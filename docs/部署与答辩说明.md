# 在线拍卖平台部署与答辩说明

## 1. 文档目的

本文档对应 `S15`，用于说明在线拍卖平台在课程设计交付阶段的本地部署、演示初始化、演示账号、答辩展示顺序和最终回归验证方式。

本步骤不引入新的业务架构，继续复用现有 `CMake + CTest + MySQL + C++20` 工程基线。MySQL 仍是拍卖、出价、订单、支付等交易事实的唯一权威数据源，Redis 仍只作为可替换缓存能力。

## 2. 环境要求

建议环境如下：

- WSL2 Ubuntu 24.04
- `zsh` 或兼容 Bash 的 Shell
- CMake、GCC 或 Clang，支持 C++20
- MySQL 8 客户端与 `mysqld`
- `ctest`、`curl`、`openssl`
- 可选：Drogon、Nginx、Redis

本仓库的演示初始化脚本会使用用户态本地 MySQL 数据目录，不需要 `sudo`。若本机没有 Drogon，应用会进入 bootstrap fallback 模式，可完成配置、数据库与测试验证，但不会长驻提供 HTTP 服务。

## 3. 部署与初始化

### 3.1 构建工程

```bash
./scripts/bootstrap.sh
```

该命令会创建 `build/`、`logs/`、`data/uploads/`，执行 CMake 配置与构建。

### 3.2 初始化演示环境

```bash
./scripts/deploy/init_demo_env.sh
```

脚本执行内容如下：

- 构建当前工程。
- 初始化用户态本地 MySQL，默认库名为 `auction_demo`。
- 导入 `sql/schema.sql`、`sql/seed.sql` 和 `sql/demo_data.sql`。
- 生成演示配置 `build/demo_config/app.demo.json`。
- 创建演示图片占位文件到 `data/uploads/demo/`。
- 执行 `auction_app --check-config` 和 `auction_app --check-db`。

可选环境变量如下：

```bash
AUCTION_DEMO_DB_NAME=auction_demo
AUCTION_DEMO_DB_USER=auction_user
AUCTION_DEMO_DB_PASSWORD=change_me
```

### 3.3 启动演示服务

```bash
./scripts/deploy/run_demo_server.sh
```

应用端口为 `127.0.0.1:18080`。若使用 Nginx 模板，`config/nginx.conf` 的本地演示入口为 `http://127.0.0.1:18081`，并反向代理到 `127.0.0.1:18080`。

浏览器前端演示入口：

```text
http://127.0.0.1:18080/demo
```

健康检查命令：

```bash
curl http://127.0.0.1:18080/healthz
```

演示数据接口：

```bash
curl http://127.0.0.1:18080/api/demo/dashboard
```

## 4. 演示账号

`sql/demo_data.sql` 会插入或更新以下演示账号，密码仅用于本地答辩演示，不用于生产环境：

| 用户名 | 密码 | 角色 | 用途 |
|---|---|---|---|
| `admin` | `Admin@123` | 管理员 | 审核、拍卖配置、统计与异常处理展示 |
| `support` | `Support@123` | 运维/客服 | 日志、通知、异常辅助处理展示 |
| `seller_demo` | `Seller@123` | 普通用户 | 卖家发布、成交、评价展示 |
| `buyer_demo` | `Buyer@123` | 普通用户 | 买家出价、支付、评价展示 |
| `buyer_demo_2` | `BuyerB@123` | 普通用户 | 竞争出价和被超越提醒展示 |

账号密码以 SHA-512 crypt 哈希形式写入数据库，不保存明文密码。

## 5. 演示数据

演示数据位于 `sql/demo_data.sql`，支持重复导入。脚本会清理标题以 `演示` 开头的旧拍品和 `DEMO-%` 相关交易数据，再插入新的演示闭环。

当前演示数据覆盖：

- 已成交拍品：包含出价、订单、支付、支付回调、双方评价和统计数据。
- 竞拍中拍品：包含两名买家的竞争出价、当前最高价和最高出价者。
- 已流拍拍品：用于展示无人出价后的流拍状态。
- 待开始拍品：用于展示拍卖配置和调度前状态。
- 站内通知：包含支付成功、订单已支付和出价被超越提醒。
- 运维数据：包含 `task_log`、`operation_log` 和 `statistics_daily` 演示记录。

前端演示台会从 `/api/demo/dashboard` 读取上述数据，并在浏览器中展示主流程、角色、拍卖状态、订单支付、评价、高风险链路和运维证据。

## 6. 答辩展示顺序

建议按以下顺序展示：

1. 需求边界：校园或小型社区在线拍卖平台，覆盖拍品、审核、竞价、支付、评价、统计和运维。
2. 架构方案：C++20 模块化单体，MySQL 作为交易事实源，缓存和通知不决定最终一致性。
3. 部署初始化：执行 `./scripts/deploy/init_demo_env.sh`，展示配置检查和数据库检查结果。
4. 运行入口：执行 `./scripts/deploy/run_demo_server.sh`，在具备 Drogon 的环境展示 `/healthz`。
5. 前端演示台：打开 `http://127.0.0.1:18080/demo`，点击“播放演示顺序”。
6. 主业务闭环：围绕演示数据说明拍品审核、拍卖创建、竞价、成交、订单、支付、履约和评价。
7. 高风险链路：说明并发出价、结束竞争、支付回调幂等、缓存降级和通知失败不回滚。
8. 测试证据：展示 `scripts/test.sh frontend`、`scripts/test.sh risk`、全量 `ctest` 和 `docs/测试报告.md`。
9. 运维可观测：展示 `operation_log`、`task_log`、`notification` 和 `statistics_daily` 的数据闭环。

也可以直接查看文字版演示提纲：

```bash
./scripts/deploy/show_demo_walkthrough.sh
```

## 7. 最终验证

最终交付前执行：

```bash
./scripts/deploy/verify_release.sh
```

该脚本会顺序执行：

- 演示环境初始化。
- 前端演示数据验证 `scripts/test.sh frontend`。
- 高风险专项回归 `scripts/test.sh risk`。
- 全量 `ctest --test-dir build --output-on-failure`。

若只需要分步验证，可执行：

```bash
cmake -S . -B build
cmake --build build
./scripts/deploy/init_demo_env.sh
./scripts/test.sh frontend
./scripts/test.sh risk
ctest --test-dir build --output-on-failure
```

## 8. 已知边界

- 当前仓库已补充用于答辩展示的只读前端演示台；完整业务 HTTP 控制器批量接入仍不在当前追加范围内。
- 当前 Redis 为可替换旁路能力，高风险验证使用进程内缓存失败注入证明“缓存失败不破坏交易事实”。
- 当前支付为 mock 支付适配器与签名校验闭环，不连接真实第三方支付平台。
- 当前本地 MySQL 由用户态脚本启动；若存在残留 `mysqld` 占用 `build/test_mysql`，需要先停止残留进程后再重试。
