# 在线拍卖平台部署与答辩说明

## 1. 文档目的

本文档用于说明在线拍卖平台在课程设计交付阶段的本地部署、演示初始化、演示账号、真实业务前端入口、答辩展示顺序和最终回归验证方式。历史上的 `/demo` 只读展示入口仅作为旧交付形态说明，不再属于当前主系统入口。

本步骤不引入新的业务架构，继续复用现有 `CMake + CTest + MySQL + C++20` 工程基线。MySQL 仍是拍卖、出价、订单、支付等交易事实的唯一权威数据源，Redis 仍只作为可替换缓存能力。

## 2. 环境要求

建议环境如下：

- WSL2 Ubuntu 24.04
- `zsh` 或兼容 Bash 的 Shell
- CMake、GCC 或 Clang，支持 C++20
- MySQL 8 客户端与 `mysqld`
- `ctest`、`curl`、`openssl`
- 可选：Drogon、Nginx、Redis

本仓库的演示初始化脚本会使用用户态本地 MySQL 数据目录，不需要 `sudo`。若本机没有 Drogon，应用会进入 bootstrap fallback 模式，可完成配置、数据库与测试验证，但不会长驻提供 HTTP 服务。

## 3. 部署与初始化

### 3.1 构建工程

```bash
./scripts/bootstrap.sh
```

该命令会创建 `build/`、`logs/`、`data/uploads/`，执行 CMake 配置与构建。

### 3.2 当前可执行初始化与验收入口

```bash
./start.sh --config config/app.local.json
```

当前仓库中真实存在并推荐使用的入口如下：

- `./start.sh --config config/app.local.json`
  - 当前一键启动入口
  - 会先执行 `--check-config` 和 `--check-db`
  - 数据库不可用时会自动尝试 `scripts/db/setup_local_mysql.sh`
  - 该 fallback 使用 `build/local_mysql/runtime` 作为持久数据目录，重启后保留拍卖、出价、订单、支付等业务历史
  - 默认同时启动后端 `18080` 和前端开发服务 `3000`
- `./scripts/deploy/verify_release.sh`
  - 当前最终发布验收入口
  - 会自动初始化本地测试 MySQL、启动后端、校验健康检查并串联 `frontend`、`http`、`risk` 与全量 `ctest`
  - 该验证入口继续使用 `build/test_mysql/run-*` 临时测试库，不复用 `start.sh` 的持久运行库

历史说明：

- 旧答辩阶段曾存在 `scripts/deploy/init_demo_env.sh`、`run_demo_server.sh`、`show_demo_walkthrough.sh`
- 当前仓库中这些脚本已不存在，因此不再作为当前可执行流程
- 旧 `auction_demo`、`sql/demo_data.sql`、`build/demo_config/app.demo.json` 和 `data/uploads/demo/` 仅属于历史交付口径，不再属于当前仓库交付物

### 3.3 当前浏览器入口

```bash
./start.sh --config config/app.local.json
```

默认会输出两个地址：

- 前端开发入口：`http://127.0.0.1:3000`
- 后端服务入口：`http://127.0.0.1:18080`

补充说明：

- 当前用户实际操作入口优先以 `frontend/` 开发服务为准，即 `http://127.0.0.1:3000`
- 后端仍保留 `/healthz` 和真实 `/api/...` 接口
- 代码中仍有历史 `/app` 路由兼容层，但当前仓库未保留对应 `assets/app/` 静态文件，因此 `/app` 不再作为当前主验收入口

当前浏览器入口：

```text
http://127.0.0.1:3000
```

健康检查命令：

```bash
curl http://127.0.0.1:18080/healthz
```

## 5. 演示数据

当前演示与联调数据主要来自 `sql/schema.sql` + `sql/seed.sql`，以及 HTTP/集成测试过程中生成的本地测试数据。历史 `sql/demo_data.sql` 已不在当前仓库中。

当前演示与联调通常覆盖：

- 已成交拍品：包含出价、订单、支付、支付回调、双方评价和统计数据。
- 竞拍中拍品：包含两名买家的竞争出价、当前最高价和最高出价者。
- 已流拍拍品：用于展示无人出价后的流拍状态。
- 待开始拍品：用于展示拍卖配置和调度前状态。
- 站内通知：包含支付成功、订单已支付和出价被超越提醒。
- 运维数据：包含 `task_log`、`operation_log` 和 `statistics_daily` 演示记录。

当前主展示入口为 `frontend/` 启动后的真实业务前端工作台，默认地址 `http://127.0.0.1:3000`。它通过真实 `/api/...` 与 `/ws/...` 接口展示认证、卖家拍品、管理员审核建拍、公开拍卖、出价、订单支付、履约、评价、通知、统计和运维页面。历史 `/demo` 只读展示台和 `/api/demo/dashboard` 仅保留在旧文档记录中。

## 6. 答辩展示顺序

建议按以下顺序展示：

1. 需求边界：校园或小型社区在线拍卖平台，覆盖拍品、审核、竞价、支付、评价、统计和运维。
2. 架构方案：C++20 模块化单体，MySQL 作为交易事实源，缓存和通知不决定最终一致性。
3. 部署初始化：执行 `./start.sh --config config/app.local.json`，展示配置检查、数据库检查和自动 fallback 结果。
4. 健康检查：在具备 Drogon 的环境展示 `curl http://127.0.0.1:18080/healthz`。
5. 真实业务前端：打开 `http://127.0.0.1:3000`，使用演示账号登录，分别展示管理员、卖家、买家和客服/运维入口。
6. 主业务闭环：围绕演示数据说明拍品审核、拍卖创建、竞价、成交、订单、支付、履约和评价。
7. 高风险链路：说明并发出价、结束竞争、支付回调幂等、缓存降级和通知失败不回滚。
8. 测试证据：展示 `scripts/test.sh frontend`、`scripts/test.sh http`、`scripts/test.sh risk`、全量 `ctest` 和 `docs/测试报告.md`。
9. 运维可观测：展示 `operation_log`、`task_log`、`notification` 和 `statistics_daily` 的数据闭环。

当前仓库未保留独立“文字版演示提纲脚本”；答辩演示步骤以本文件和 `docs/答辩演示操作手册.md` 为准。

## 7. 最终验证

最终交付前执行：

```bash
./scripts/deploy/verify_release.sh
```

该脚本会顺序执行：

- 前端 `release-check`。
- 真实后端健康检查与真实业务接口冒烟。
- 前端真实业务工作台验证 `scripts/test.sh frontend`。
- 真实 HTTP 基线回归 `scripts/test.sh http`。
- 高风险专项回归 `scripts/test.sh risk`。
- 全量 `ctest --test-dir build --output-on-failure`。

若只需要分步验证，可执行：

```bash
cmake -S . -B build
cmake --build build
./scripts/test.sh frontend
./scripts/test.sh http
./scripts/test.sh risk
ctest --test-dir build --output-on-failure
```

当前仍可复现的部署/回归结论如下：

- `scripts/deploy/verify_release.sh` 通过，前端构建检查、高风险专项和全量 `ctest` 均通过。

- 后端健康检查入口为 `curl http://127.0.0.1:18080/healthz`。

## 8. 已知边界

- 当前仓库已按 `S17-S30` 完成真实业务前端接入、发布门禁接入、最终回归和 handoff。
- 当前 Redis 为可替换旁路能力，高风险验证使用进程内缓存失败注入证明“缓存失败不破坏交易事实”。
- 当前支付为 mock 支付适配器与签名校验闭环，不连接真实第三方支付平台。
- 当前 `start.sh` 的本地 MySQL fallback 会复用 `build/local_mysql/runtime`，用于保留本地业务历史；测试脚本和发布验证仍使用 `build/test_mysql/run-*` 临时库。
- 若存在残留 `mysqld` 占用 `build/test_mysql` 或 `build/local_mysql/runtime`，需要先停止对应残留进程后再重试。
- 当前未引入浏览器驱动截图、真实点击流、前端性能压测或 200 并发在线验证；这些属于课程设计交付之外的可选增强。
