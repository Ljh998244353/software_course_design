# 在线拍卖平台前端模块说明

## 1. 文档目的

本文档说明当前仓库前端相关实现的真实入口、代码位置、验证方式和历史边界。

当前仓库已不再保留 `/demo` 只读演示台作为主入口。当前唯一产品化前端主入口为：

- `frontend/` 启动后的 Next.js 页面，默认访问地址为 `http://127.0.0.1:3000`

历史上的 `/demo`、`/api/demo/dashboard`、`assets/demo/` 和对应只读演示服务，仅属于旧答辩交付形态说明，不再属于当前主链路。

## 2. 当前入口与能力

当前前端相关能力如下：

- 真实前端目录：`frontend/`
- 主入口页面：`frontend/app/`
- 后端 API：`/api/...`
- 实时通道：`/ws/auction/{id}`
- 前端验证入口：`scripts/test.sh frontend`
- HTTP 回归入口：`scripts/test.sh http`
- 高风险回归入口：`scripts/test.sh risk`
- 发布门禁入口：`scripts/deploy/verify_release.sh`

当前真实前端已接入的业务能力包括：

- 注册、登录、登出、会话恢复
- 角色导航与路由守卫
- 卖家拍品发布、编辑、图片元数据、提交审核
- 管理员待审列表、审核、建拍、拍卖管理
- 买家拍卖大厅、详情、出价、通知轮询
- 订单详情、测试支付、履约、评价
- 管理统计、运维异常、日志与补偿入口

## 3. 当前代码位置

前端代码：

- `frontend/app/`
- `frontend/components/`
- `frontend/lib/`
- `frontend/types/`

后端 HTTP / WS 接入层：

- `src/access/http/`
- `src/ws/`

统一前端验证脚本：

- `scripts/test_frontend.sh`
- `scripts/test.sh`
- `scripts/deploy/verify_release.sh`

页面级合同测试：

- `tests/ui/app_page_acceptance_tests.cpp`

真实 HTTP 回归：

- `scripts/test_http.sh`

## 4. 当前验证方式

前端静态与类型检查：

```bash
scripts/test.sh frontend
```

该入口会进入 `frontend/` 执行：

```bash
npm run release-check
```

真实 HTTP 回归：

```bash
scripts/test.sh http
```

高风险回归：

```bash
scripts/test.sh risk
```

全量测试：

```bash
ctest --test-dir build --output-on-failure
```

最终发布验证：

```bash
scripts/deploy/verify_release.sh
```

当前发布门禁已收敛为：

- `frontend`
- `http`
- `risk`
- `ctest`

不再要求 `/demo` 或 `api/demo/dashboard` 作为当前主验收前提。

## 5. 历史边界

以下内容仅用于理解历史交付形态，不应再当成当前实现：

- `/demo` 只读答辩演示台
- `/api/demo/dashboard`
- `assets/demo/`
- `src/access/http/demo_http.*`
- `src/application/demo_dashboard_service.*`
- `sql/demo_data.sql`
- `tests/demo/demo_dashboard_tests.cpp`

这些历史路径在当前仓库中已删除或退出主链路。后续若文档和代码不一致，应以当前代码现实为准。
