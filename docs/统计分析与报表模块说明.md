# 在线拍卖平台统计分析与报表模块说明

## 1. 文档目的

本文档用于说明统计分析与报表模块在课程设计阶段的职责范围、日报口径、查询与导出契约、任务重跑方式和验证结果。

本模块位于订单、支付与评价模块之后，负责把真实交易事实聚合为管理端可直接查看的 `statistics_daily` 日报表。
`S25` 已在原 `S10` 服务层能力之上接入 Drogon HTTP 控制器和 `/app` 管理端统计页面。

## 2. 模块职责

本模块负责以下能力：

- 按自然日聚合 `auction`、`bid_record`、`order_info`、`payment_record` 等真实事实表
- 将日报结果 upsert 到 `statistics_daily`，支持同一天重复重跑覆盖
- 提供管理员查询日报表能力
- 提供管理员导出 CSV 报表能力
- 为统计任务写入 `task_log`，为重跑与导出写入 `operation_log`
- 提供 `/api/admin/statistics/daily`、`/api/admin/statistics/daily/rebuild`、`/api/admin/statistics/daily/export` 真实 HTTP 接口
- 在 `/app` 管理员页面展示日报汇总、日报列表、重跑入口和 CSV 导出预览

本模块不负责以下内容：

- 不负责实时大屏、复杂图表或多维钻取分析
- 不负责对象存储落盘导出文件，本步仅输出 CSV 内容
- 不负责任务重试编排、异常补偿入口和统一监控告警
- 不负责修改主交易链路事实，只消费既有业务表

## 3. 核心实体与依赖

### 3.1 核心实体

- `Auction`
- `BidRecord`
- `OrderInfo`
- `PaymentRecord`
- `StatisticsDaily`

### 3.2 主要依赖

| 依赖对象 | 作用 | 边界说明 |
|---|---|---|
| 认证与权限模块 | 校验管理端查询与导出权限 | 只有 `ADMIN` 可查看和导出日报 |
| `auction` 表 | 提供拍卖结束日和流拍状态 | 日报按 `end_time` 自然日归档活动 |
| `bid_record` 表 | 提供日出价次数 | 按 `bid_time` 自然日统计 |
| `order_info` 表 | 提供订单状态与成交金额 | `CLOSED` 不计入成交 |
| `payment_record` 表 | 提供支付成功事实 | 只有存在成功支付才计入成交数与 GMV |
| `statistics_daily` 表 | 保存日报聚合结果 | 同一 `stat_date` 唯一 |
| `task_log` / `operation_log` | 保存任务执行与管理操作审计 | 供后续运维模块消费 |

## 4. 统计口径

### 4.1 统计日定义

- `stat_date` 使用 `YYYY-MM-DD`
- 活动类指标按 `auction.end_time` 所在自然日归档
- 出价次数按 `bid_record.bid_time` 所在自然日归档

### 4.2 字段口径

| 字段 | 口径 |
|---|---|
| `auction_count` | 当日已到结束时间的拍卖活动数 |
| `sold_count` | 当日结束活动中，关联订单未关闭且存在成功支付记录的数量 |
| `unsold_count` | 当日结束活动中，`auction.status = UNSOLD` 或关联订单已关闭的数量 |
| `bid_count` | 当日出价记录数 |
| `gmv_amount` | 当日结束活动中已成功支付订单的 `final_amount` 汇总 |

说明：

- 因为 `sold_count` 依赖支付成功事实，而 `unsold_count` 依赖流拍或超时关闭结果，所以在“支付仍待完成”的中间态下，`sold_count + unsold_count` 可能小于 `auction_count`
- `statistics_daily` 是聚合缓存表，不是交易事实源；若历史事实修正，可按日重跑覆盖

## 5. 业务流程

### 5.1 日报重跑流程

1. 调度任务或后台入口传入 `stat_date`
2. 系统校验日期格式并开启事务
3. 聚合当日结束活动数、成交数、流拍数、出价数和 GMV
4. 将结果 upsert 到 `statistics_daily`
5. 写入 `operation_log` 和 `task_log`
6. 后续同一天再次重跑时覆盖旧值，不新增第二行

### 5.2 查询日报流程

1. 管理员携带 Bearer Token 调用日报查询接口
2. 系统校验 `ADMIN` 角色与日期范围
3. 按日期升序返回 `statistics_daily` 中已有日报记录

### 5.3 导出流程

1. 管理员携带 Bearer Token 调用导出接口
2. 系统复用日报查询结果
3. 生成 CSV 文本并返回文件名、内容类型与行数
4. 写入一次导出审计日志

## 6. 接口设计

所有接口遵循统一响应结构：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

### 6.1 `GET /api/admin/statistics/daily`

用途：

- 查询日报表

鉴权要求：

- 需要登录
- 仅 `ADMIN`

请求字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| startDate | string | 是 | 开始日期，格式 `YYYY-MM-DD` |
| endDate | string | 是 | 结束日期，格式 `YYYY-MM-DD` |

返回字段：

- `statDate`
- `auctionCount`
- `soldCount`
- `unsoldCount`
- `bidCount`
- `gmvAmount`

### 6.2 `GET /api/admin/statistics/daily/export`

用途：

- 导出日报 CSV

请求字段：

- 同 `GET /api/admin/statistics/daily`

返回字段：

- `fileName`
- `contentType`
- `csvContent`
- `rowCount`

### 6.3 `POST /api/admin/statistics/daily/rebuild`

用途：

- 按天重跑日报聚合

请求字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| statDate | string | 是 | 目标统计日，格式 `YYYY-MM-DD` |

## 7. 关键错误码

| 错误码 | 含义 | 触发场景 |
|---|---|---|
| `4108` | `auth permission denied` | 非管理员查询或导出 |
| `4801` | `statistics date invalid` | 日期格式不是 `YYYY-MM-DD` |
| `4802` | `statistics range invalid` | 开始日期晚于结束日期 |
