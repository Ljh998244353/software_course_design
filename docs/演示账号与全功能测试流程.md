## 2. 演示前准备

启动系统：

```bash
./start.sh --config config/app.local.json
```

确认以下地址可访问：

- 前端：`http://127.0.0.1:3000`
- 后端健康检查：`http://127.0.0.1:18080/healthz`

## 3. 角色分工

- `admin`：管理员，负责审核拍品、创建拍卖、查看统计与运维信息
- `seller_demo`：卖家，负责发布拍品
- `buyer_demo`：主买家，负责浏览、出价、中标后支付、确认收货、评价
- `viewer_demo`：第二用户，负责参与竞价，演示多用户竞争与价格变化

## 4. 全功能测试/演示流程

### Step 1：管理员确认后台可用

1. 使用 `admin / Admin@123` 登录
2. 进入管理大盘
3. 确认以下模块可以打开：
   - 待审核列表
   - 拍卖监控
   - 统计报表
   - 运维终端

演示目标：

- 管理员账号可正常登录
- 管理员页面与普通用户页面隔离

### Step 2：卖家发布拍品

1. 退出管理员
2. 使用 `seller_demo / Seller@123` 登录
3. 进入“发布拍品”
4. 按 4 步向导填写：
   - 标题
   - 描述
   - 图片 URL
   - 起拍价
   - 建议加价幅度
   - 延时保护窗口
   - 顺延秒数
5. 提交审核

建议演示数据：

- 标题：`iPad Air 5 64G 蓝色`
- 描述：`99 新，无拆修，支持当面验货`
- 图片 URL：任意可访问图片地址 1~2 张
- 起拍价：`300`
- 建议加价幅度：`20`

演示目标：

- 草稿保存正常
- 图片列表展示正常
- 提交审核成功

### Step 3：管理员审核拍品

1. 使用 `admin / Admin@123` 登录
2. 进入管理大盘待审核列表
3. 找到 `seller_demo` 刚提交的拍品
4. 打开审核抽屉查看详情
5. 点击“批准”

演示目标：

- 管理员能看到待审核拍品
- 审核动作生效
- 拍品状态从待审核流转为可建拍

### Step 4：管理员创建拍卖

1. 仍使用管理员账号
2. 在管理端找到已审核通过的拍品
3. 创建拍卖
4. 设置：
   - 开始时间
   - 结束时间
   - 起拍价
   - 最小加价幅度

演示目标：

- 管理员可完成建拍
- 拍品出现在公开拍卖大厅

### Step 5：买家浏览拍卖大厅

1. 使用 `buyer_demo / Buyer@123` 登录
2. 打开拍卖大厅
3. 查看新创建的拍卖
4. 点击进入详情页

演示目标：

- 大厅列表与真实后端联通
- 详情页可正常打开

### Step 6：多用户竞价

1. `buyer_demo` 打开同一拍卖详情页
2. `viewer_demo` 也打开同一拍卖详情页
3. `buyer_demo` 先出一口价
4. `viewer_demo` 再出更高价格
5. 观察页面价格、出价历史、最高出价人变化

演示目标：

- 多用户真实竞价可执行
- 最高价与出价历史正确刷新
- 详情页实时通道或轮询降级逻辑正常

### Step 7：等待拍卖结束并生成订单

1. 等待拍卖结束，或在管理员侧推进到结束状态
2. 确认最高价用户成为中标者
3. 用中标账号进入订单/支付页

演示目标：

- 一场拍卖只生成一笔订单
- 未中标用户不会进入支付主链路

### Step 8：中标用户支付

1. 使用中标账号登录（通常是 `buyer_demo` 或 `viewer_demo`）
2. 进入结算页
3. 选择模拟支付方式
4. 完成支付

演示目标：

- 本地 mock 支付闭环正常
- 订单状态进入已支付

### Step 9：卖家发货

1. 使用 `seller_demo / Seller@123` 登录
2. 进入订单视图
3. 对已支付订单执行发货

演示目标：

- 卖家履约动作生效
- 订单状态推进

### Step 10：买家确认收货

1. 使用中标买家账号登录
2. 进入订单视图
3. 执行确认收货

演示目标：

- 收货动作生效
- 订单进入完成态

### Step 11：买家评价

1. 仍使用中标买家账号
2. 对已完成订单提交评价

演示目标：

- 评价功能闭环可用
- 信用与评价链路打通

### Step 12：管理员查看统计与运维

1. 使用 `admin / Admin@123` 登录
2. 打开管理大盘
3. 查看：
   - 统计日报
   - 拍卖监控
   - 运维日志/异常面板

演示目标：

- 管理端可看到业务结果沉淀
- 从发布、审核、竞价、成交、支付、履约到评价的全链路闭环完成

## 5. 最短答辩演示顺序

若时间有限，建议使用以下最短顺序：

1. 管理员登录，展示管理大盘
2. 卖家发布拍品
3. 管理员审核并建拍
4. 买家进入大厅和详情页
5. 买家与第二用户竞价
6. 中标用户支付
7. 卖家发货
8. 买家收货并评价
9. 管理员查看统计结果
