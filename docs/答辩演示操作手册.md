# 在线拍卖平台答辩演示操作手册

## 1. 演示目标

本手册用于现场答辩时快速复现在线拍卖平台功能。推荐演示方式是“浏览器前端演示台 + 数据库事实 + 自动化测试证据”。

演示重点：

- 系统能启动并提供浏览器入口。
- 页面数据来自 MySQL 演示库，不是静态截图。
- 主流程覆盖拍品、审核、拍卖、出价、订单、支付、评价、统计和运维。
- 高风险链路有自动化测试证明。

## 2. 演示前准备

进入项目目录：

```bash
cd /home/ljh/project/soft_course_design
```

初始化演示环境：

```bash
./scripts/deploy/init_demo_env.sh
```

如果该命令提示 MySQL 数据文件被锁定，通常是之前残留的用户态 `mysqld` 占用 `build/test_mysql/data`。先检查：

```bash
pgrep -af '/home/ljh/project/soft_course_design/build/test_mysql'
```

只处理本项目残留进程，不要误杀系统 MySQL。

## 3. 启动服务

在第一个终端启动后端服务：

```bash
./scripts/deploy/run_demo_server.sh
```

服务会监听：

```text
http://127.0.0.1:18080
```

在第二个终端检查健康状态：

```bash
curl -i http://127.0.0.1:18080/healthz
```

预期结果：

- HTTP 状态为 `200 OK`
- 响应中包含 `status=ok`
- 响应中包含 `drogonEnabled=true`

## 4. 打开前端演示台

浏览器打开：

```text
http://127.0.0.1:18080/demo
```

页面加载后，先确认顶部摘要卡片出现演示账号数、拍品数、拍卖数、出价数、订单数和成交金额。

如果页面提示数据加载失败，按顺序检查：

1. 是否已经执行 `./scripts/deploy/init_demo_env.sh`
2. `./scripts/deploy/run_demo_server.sh` 是否仍在运行
3. `curl http://127.0.0.1:18080/api/demo/dashboard` 是否返回 `code=0`

## 5. 推荐讲解顺序

### 5.1 需求和架构

先说明项目背景：

- 面向校园或小型社区的在线拍卖平台。
- 覆盖发布、审核、竞价、结算、评价、统计和运维。
- 后端采用 `C++20 + Drogon + MySQL` 的模块化单体。

再说明关键架构约束：

- MySQL 是拍卖、出价、订单、支付等交易事实的唯一权威数据源。
- Redis 或进程内缓存只做热点读取和旁路优化，不决定最终一致性。
- 通知和统计不阻塞出价、订单、支付等核心链路。

### 5.2 浏览器主流程

在 `/demo` 页面点击“播放演示顺序”，按页面滚动依次讲：

1. 主业务流程：从拍品发布到评价闭环。
2. 演示账号：管理员、运维/客服、卖家、买家、竞争买家。
3. 拍卖现场：已成交、竞拍中、已流拍、待开始。
4. 订单支付：展示 `DEMO-ORDER-PAID` 的成交金额、支付状态和评价。
5. 高风险链路：说明并发、幂等、缓存降级和安全负向测试。
6. 数据证据：展示出价、通知、任务日志、操作日志和统计日报。

### 5.3 数据库事实证明

可在终端展示接口数据：

```bash
curl http://127.0.0.1:18080/api/demo/dashboard
```

也可以直接查询 MySQL：

```bash
mysql --socket=build/test_mysql/mysql-runtime.sock -uroot auction_demo
```

进入 MySQL 后执行：

```sql
SELECT i.title, i.item_status, a.status, a.current_price
FROM item i
JOIN auction a ON a.item_id = i.item_id
WHERE i.title LIKE '演示%';

SELECT order_no, final_amount, order_status, paid_at, completed_at
FROM order_info
WHERE order_no LIKE 'DEMO-%';
```

讲解重点是：页面上的拍卖状态、最高价和订单状态都能在数据库中对应。

## 6. 测试证据展示

前端演示数据专项验证：

```bash

```

高风险专项测试：

```bash
scripts/test.sh risk
```

全量测试：

```bash
ctest --test-dir build --output-on-failure
```

讲解口径：

- `frontend` 校验演示接口能从 `auction_demo` 读取完整页面数据。
- `risk` 覆盖并发出价、拍卖结束竞争、支付回调幂等、缓存降级、通知失败和安全负向路径。
- 全量 `ctest` 覆盖工程冒烟、数据库、模块流、集成流和专项测试。

## 7. 常见问题处理

### 7.1 浏览器出现 404

如果访问的是：

```text
http://127.0.0.1:18080/api/items
```

该地址直接用浏览器访问时是 `GET /api/items`，当前只注册了 `POST /api/items` 创建拍品和 `GET /api/items/mine` 我的拍品列表，因此直接打开可能返回统一 JSON `404`。

答辩时只读总览应访问：

```text
http://127.0.0.1:18080/demo
```

真实业务操作应访问：

```text
http://127.0.0.1:18080/app
```

### 7.2 浏览器无法连接

如果浏览器提示连接失败，说明服务没有启动或端口不对。重新执行：

```bash
./scripts/deploy/run_demo_server.sh
```

再检查：

```bash
curl -i http://127.0.0.1:18080/healthz
```

### 7.3 页面打开但没有数据

重新初始化演示数据：

```bash
./scripts/deploy/init_demo_env.sh
```

然后重启服务并刷新页面。
